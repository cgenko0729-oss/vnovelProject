# 《辣妹与宅宅》剧本包 —— 素材与资产登记清单

这一包剧本是**教学 + 可直接续写的正式骨架**。所有 id 都用了语义化真名，
下面列出需要你在 Unity 里补齐的登记项。**没补齐的部分会在 Console 报警告但不会中断播放**
（背景找不到就保持原背景、bgm/se 找不到就静音），所以可以边补素材边试跑。

---

## 一、章节登记：已经自动化，不用再拖了 ✅

以前必须手动把每个 `.vn.txt` 拖进 `VNScriptRunner.Chapters`，否则跨文件跳转全部报错，
而且**重建场景后会全部丢失**。现在生成器会自动扫描 `Assets/Scenarios/*.vn.txt` 登记全部章节。

你只需要设一次**入口剧本**，而且是设在资产里（永不丢失）：

> **Tools → VN Effects → Game Config → Create or Select**
> → 在 `VNGameConfig` 的 `Entry Script` 字段设为 `第1章.vn.txt`

新增剧本文件后，跑一次 **Game Config → Rescan Asset Folders** 或重建场景即可。

> 详见下面第十节「零拖拽工作流」。

## 九、文件关系图

```
第1章  ──chapter──▶  主循环 ◀──jump──┐
                       │             │
        每月初 call ────┼──▶ 节日活动集（固定月份事件，内含文化祭周日程）
                       ├──▶ 好感事件_星野结衣（好感阈值阶梯事件）
                       └──▶ 公共片段（送礼/转场/结算等带参数子程序）
                       │
        月份+好感门槛 jump ──▶ 第2章 / 第3章 ──jump 回──▶ 主循环::行动菜单
                       │
        剩余月数<=0 jump ──▶ 结局集::结局判定 ──▶ 六种结局
```

**为什么这么分**：`chapter` 会清空调用栈（适合"彻底换幕"），`jump 文件::标签` 保留调用栈
（适合"去别处演一段再回来"），`call 文件::标签` 则会自动回到原处。三者的取舍在每个
文件头的注释里都写了，剧本正文里也标了"这里为什么用这个而不是那个"。

## 十、零拖拽工作流 ★（重建场景不再丢配置）

### 问题
`Tools → VN Effects → Create Script Demo Scene` 内部会
`EditorSceneManager.NewScene(EmptyScene)`——**丢弃当前场景从零重造**。
所以挂在场景组件上的一切人工绑定（背景库、音频库、章节列表、地图坐标、入口剧本）
每次重建全部清空。这不是操作失误，是生成器的定位使然（它本来是"重建 demo"用的）。

### 解法
所有内容绑定搬进 **`Assets/Resources/VNGameConfig.asset`**（一个 ScriptableObject）。
它在 `Resources` 下，运行时自动加载，**场景里一个引用都不需要**。

### 三个菜单

| 菜单 | 什么时候用 |
|---|---|
| **Game Config → Create or Select** | 第一次；之后用来打开资产改背景/音频 |
| **Game Config → Import From Scene** | **一次性搬家**：把你现在场景里已经配好的东西全部抓进资产 |
| **Game Config → Rescan Asset Folders** | 新增了角色/剧本/CG/属性/商店/日程/任务资产之后 |

### 首次迁移步骤（照做一遍就永久解决）

1. 打开你**现在配好的**场景（先别重建！）
2. **Tools → VN Effects → Game Config → Import From Scene**
   → 弹窗会列出导入了多少条背景/音频/章节/地图地点
3. **Tools → VN Effects → Game Config → Rescan Asset Folders**
   → 自动登记全部角色/章节/CG/属性/商店/日程/任务资产
4. 打开 `Assets/Resources/VNGameConfig.asset`，把 `Entry Script` 设为 `第1章.vn.txt`
5. 现在再跑 **Create Script Demo Scene** —— 所有东西会自动回到位

### 之后的分工

| 数据 | 维护方式 |
|---|---|
| 背景 id→图、音频 id→clip+音量、地图坐标、入口剧本 | **手工填在 VNGameConfig 资产里**（一次，永久） |
| 角色 / 章节剧本 / CG / 属性 / 商店 / 日程 / 任务 | **自动扫目录**，丢进对应文件夹即可，无需登记 |
| 场景组件上的 `backgrounds` / `bgmLibrary` 等 | **别再填了**；资产里非空就会覆盖它们 |

### 覆盖规则（只有一条）
> VNGameConfig 里**填了的**项覆盖场景设置，**留空的**项保持场景原样。

所以想临时在场景里试别的东西，把资产里对应项清空即可，不必删资产。
