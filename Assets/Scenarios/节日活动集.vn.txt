# ============================================================
# 节日活动集.vn.txt —— 固定月份事件（文件头路由模式）
# ------------------------------------------------------------
# 【进入方式】主循环每个月初 call 一次：
#   call 节日活动集::月度路由
#
# 【本文件教学的机制】★ 路由表模式
#   一个 label 当"总机"，按 月份 分派到具体节日，没有匹配就直接 return。
#   主循环只写一行 call，从此**加节日不用碰主循环**——
#   这是整个剧本架构里性价比最高的一个模式。
#
#   为什么路由要放在被调用文件里，而不是主循环里？
#     主循环写 `if 月份==7 jump ...` 会让主循环随节日数量膨胀；
#     而且节日之间的优先级、互斥（同月两个节日）只有本文件最清楚。
#
# 【还教了】
#   event plan 周日程排程（9 月文化祭）—— 排格 → op:next 逐格派发 → rand 分级 → result 结算
#   守卫 flag 防重复（虽然每月只过一次，但读档/调试重播时会救你）
#   嵌套 call（月度路由 → 文化祭周 → 公共片段::换场）
# ============================================================


# ------------------------------------------------------------
# 月度路由 —— 总机
# ------------------------------------------------------------

# 守卫写在条件里（`&& !xx已办`）而不是事件内部：

# 这个月没有节日
return

# ============================================================
# 九月：文化祭 —— event plan 周日程排程 ★本包玩法密度最高的一段
# ============================================================
# 【plan 的四块拼图】
#   ① event plan slots: pool:        排程面板 → 写 flag 日程_1..N / 日程数 / 当前格=0
#   ② event plan op:next             逐格派发 → 当前格+1，行动编号写进 flag 当前行动
#                                     超出格数返回结果 end
#   ③ flag 运气 rand:1-100           掷骰
#   ④ event result grade:            结算弹窗（fail/normal/good/great）
#   **概率表写在剧本里**，因为它属于"内容"而不是"逻辑"——
#   想调平衡直接改这里的数字，不用碰一行 C#。
#
# 【行动编号】pool 里的名字按出现顺序编号：
#   1 布置会场 / 2 排练 / 3 做料理 / 4 休息（空格 = 0，当休息处理）
# 【升级路径】想要图标 + 收益文案，就建一个 planId=文化祭准备 的 VNPlanDef 资产
#   （见 README）。找不到该资产时会自动退化成 pool 编号模式，现在就能跑。
# ============================================================
