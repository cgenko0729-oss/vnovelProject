# ============================================================
# 第3章.vn.txt —— 三月：天台，和那句说不出口的话
# ------------------------------------------------------------
# 【进入方式】主循环 月初（3 月 = 时间线上的最后一个月，日本的转学/毕业季）：
#   if 月份==3 && 好感度_结衣>=55 && 任务_告白大作战>=3 && !第3章已看 jump 第3章::序
#   ↑ 四重门槛的示范：时间 + 数值 + **任务阶段** + 守卫。
#     `任务_告白大作战>=3` 说明任务状态本身就是 flag，可以直接当剧情门槛用，
#     不需要额外再开一个变量。
#
# 【本文件教学的机制】
#   多层嵌套条件（&& || ! 与括号混用）
#   event qte 高难度 + 结果影响后续（不是立刻分支，而是记账给结局用）
#   choice 的 cost:（花钱买"底气"）
#   camseq 多段路径镜头做名场面
#   sakura / shake / letterbox / cg / weather 的高潮编排
#   flag 表白过 / 表白结果 —— 把"高潮的结果"存成变量，交给结局集去演
#
# 【设计心法】★
#   高潮章节**不要直接演结局**。它只负责：
#     把玩家这一年的所有积累，浓缩成一个变量（这里是 表白结果）。
#   结局的演出统一放在 结局集.vn.txt——这样加新结局时不用碰这一章。
# ============================================================

# ★ 这里演示"多条件复合判断"的实战写法：

#   （只有 choice 选项行的 if: 必须无空格。）

# ---------- 花钱买底气：cost: 的戏剧化用法 ----------
label 天台前
星野结衣 沮丧: 那……我先走了。今天值日是你吧？
: （她转身，走向门口。）
: （——三步。两步。一步。）

# ★ cost: 的妙用：不只是"买东西"，也可以是"用一年攒的资源换一次机会"。
#   付不起 = 选项置灰但**依然可见**——让玩家看见自己的不足，这是设计意图。
#   ⚠ 选项的 flag: 只支持 `名字` / `名字+N` / `名字-N`，不能赋任意值；
#     所以"把底气清零"写成目标 label 里的一条 flag 命令（见 label 错过）。
choice
* 「等一下！」——叫住她 -> 叫住她
* 把攒了一年的钱包掏空，追出去打车追到车站 cost:金钱-600 flag:底气+1 -> 叫住她
* 什么都不做，看着她走出教室 -> 错过

# ---------- QTE：把"鼓起勇气"做成玩法 ----------
# ★ 注意这个 QTE 的结果**不立刻决定成败**，只是加进 底气。
#   为什么：如果 QTE 失败就直接坏结局，玩家会觉得"一年的养成被手速毁了"。
#   正确设计是——养成决定上限，QTE 只是最后一推。
: （说啊。说出来啊。）

* fail -> 开口
