# ============================================================
# 公共片段.vn.txt —— 带参数的可复用子程序库
# ------------------------------------------------------------
# 【本文件教学的机制】
#   call 文件::标签 名:值    调用并自动返回（对比 jump = 有去无回）
#   params 必填 可选=默认值  参数声明，必须紧跟 label
#   ${名字}                  参数占位符（台词/说话者/表情/参数值/选项/跳转都能用）
#   嵌套 call 与显式转发      call 内层 角色:${角色}
#   return                   回到 call 的下一行
#
# 【核心心法】
#   凡是"同样的演出在三个以上地方重复"的东西，就抽成这里的一段子程序。
#   剧本文件从此只写"剧情本身"，演出细节集中在这一处改。
#
# 【三条铁律】（踩过就知道疼）
#   1. params 必须是 label 后的第一条有效命令，中间不能插别的命令。
#   2. 子程序必须以 return 结尾；带着调用栈跑到文件末尾会报错停机。
#   3. 参数值不能含空格（`call X 台词:你好 世界` 会被当成两个参数）。
#      要长句子就先写进子程序里，用参数只传"选择哪一句"的开关。
#
# 【易错点】
#   ${名字} 只替换 **call 传进来的参数**，不会读 flag。
#   想用 flag 的值，只能靠 if 分流（见下面的 好感反馈 子程序）。
# ============================================================


# ------------------------------------------------------------
# 送礼 —— 消耗一个道具，换好感 + 演出
#   call 公共片段::送礼 道具:蝴蝶结发饰 好感:12
#   道具 / 好感 是必填，表情 有默认值可省略
# ------------------------------------------------------------
label 送礼
params 道具 好感 表情=害羞
se 闪光
: （深呼吸……把${道具}递过去。）
我: 那个……这个，送给你。
emote 星野结衣 Surprise
wait 0.3
星野结衣 ${表情}: 诶——？送我的？你、你怎么知道我想要这个啊……
# flag 是静默写入，用来扣道具最合适（不需要飘字打扰玩家）
flag 道具_${道具} -1
# 嵌套 call：参数不会自动继承，必须显式往下传
call 好感反馈 数值:${好感}
return


# ------------------------------------------------------------
# 好感反馈 —— 加好感 + 按"当前好感总量"给不同反应
#   这是"参数 + flag 分流"的标准配合：
#   数值 用参数传（每次不同），反应等级用 if 读 flag（参数读不到 flag）
# ------------------------------------------------------------
label 好感反馈
params 数值
stat 好感度_结衣 +${数值}
# 注意 if 判断的是"加完之后"的值
if 好感度_结衣>=70 jump 好感反馈_热恋
if 好感度_结衣>=40 jump 好感反馈_亲近
if 好感度_结衣>=15 jump 好感反馈_熟络
星野结衣 微笑: 谢啦。……你这人，比看上去有意思嘛。
return
label 好感反馈_熟络
星野结衣 微笑: 嘿嘿，谢谢你啦！明天教室见～
return
label 好感反馈_亲近
fx heartbeat on
星野结衣 害羞: ……真的谢谢。你总是这样，突然就来这么一下。
fx heartbeat off
return
label 好感反馈_热恋
fx heartbeat on
sakura
星野结衣 害羞: ……笨蛋。你再对我这么好，我可要当真了哦？
fx heartbeat off
return


# ------------------------------------------------------------
# 换场 —— 统一的"切地点"演出（背景 + 色调 + 转场 + 清特效）
#   call 公共片段::换场 背景:天台 心情:Sunset 转场:CircleWipe
#   之所以抽出来：将来想给全游戏的换场加"地名横幅"，只改这一处
# ------------------------------------------------------------
label 换场
params 背景 心情=Neutral 转场=NoiseDissolve
reset effects
bg ${背景} transition:${转场}
mood ${心情}
return


# ------------------------------------------------------------
# 属性判定 —— 通用的"属性够不够"演出
#   call 公共片段::属性判定 属性名:魅力 门槛:50
#   它自己不做分流（if 读不到 ${属性名} 这种动态 flag 名），
#   只负责演出；真正的分流交给调用方 —— 这就是"子程序只管演，剧本管逻辑"。
# ------------------------------------------------------------
label 属性判定
params 属性名 门槛
: （${属性名}……要求是 ${门槛} 点吗。）
se 翻页
return


# ------------------------------------------------------------
# 月末结算 —— 主循环每个月底都会 call 一次
#   演示：子程序内部照样能用 if / jump / choice，
#   只要最终走到 return 就行（jump 不会破坏调用栈，chapter 才会）
# ------------------------------------------------------------
label 月末结算
call 换场 背景:我的房间 心情:Night 转场:Blinds
: （躺在床上，回想这个月。）
if 压力>=80 jump 月末_过劳
if 压力>=50 jump 月末_疲惫
: （状态还不错。明天也加油吧。）
jump 月末_收尾

label 月末_疲惫
: （有点累了……下个月安排点休息的时间吧。）
jump 月末_收尾

label 月末_过劳
fx dutch on
shake light
: （眼前发黑……不行，再这样下去会垮掉的。）
# 过劳惩罚：强制掉魅力，并把压力削一半（用 stat 让玩家看见飘字）
stat 魅力 -5
stat 压力 -40
fx dutch off
: （请了一天假，睡到了下午。）

label 月末_收尾
# 任务推进也放在这里统一管：好感到线就自动推进"告白大作战"的阶段
if 任务_告白大作战==1 && 好感度_结衣>=35 jump 月末_任务2
if 任务_告白大作战==2 && 好感度_结衣>=60 jump 月末_任务3
return
label 月末_任务2
quest stage 告白大作战 2
return
label 月末_任务3
quest stage 告白大作战 3
return


# ------------------------------------------------------------
# 结衣的日常一句 —— "参数当分派表"的高级技巧 ★
#   参数值不能带空格，所以"把整句台词当参数传"行不通；
#   正确做法：传一个"语气 id"，然后 **把参数直接拼进 jump 目标**——
#   jump 的目标名同样支持 ${}，于是 语气:调侃 会跳到 label 日常一句_调侃。
#   这等价于别的语言里的 switch / 函数表，一行顶一串 if。
#   call 公共片段::日常一句 语气:调侃
#   ⚠ 代价：传错名字（比如 语气:生气）会跳到不存在的 label 而报错，
#     所以只对"取值集合固定"的参数用这招。
# ------------------------------------------------------------
label 日常一句
params 语气=平常
jump 日常一句_${语气}

label 日常一句_平常
星野结衣 默认: 哟，又在看那种厚厚的书啊。
return
label 日常一句_调侃
星野结衣 坏笑: 诶～原来你会笑啊？我还以为你只会点头呢。
return
label 日常一句_关心
星野结衣 微笑: 你脸色好差诶……有好好吃饭吗？
return
