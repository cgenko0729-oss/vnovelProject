# 【进入方式】主循环 月初 的门槛跳转：
#   if 月份==6 && 好感度_结衣>=25 && !第2章已看 jump 第2章::序
#   ↑ 这行的三个条件缺一不可：
#     月份==6         什么时候演
#     好感度>=25      够不够格演（没到线就这个月不演，下个月也不演——它是限定月份的）
#     !第2章已看      **防死循环的守卫**（本章末尾会 jump 回主循环）
#
# 【本文件教学的机制】
#   进入即上锁（flag 第2章已看 1）
#   路线分歧：flag 路线 —— 一个变量代替三个布尔开关
#   choice 的隐藏选项：宅度够高才出现的"第四选项"
#   event map：地图选地点（结果名 = 地点，自动写 flag 去过_<地点>）
#   quest stage：任务阶段推进
#   cg / letterbox / weather / mood 的组合演出
#
# 【结尾去向】
#   jump 主循环::行动菜单
#   ↑ 为什么不是 ::月初？因为月初会重跑节日/好感路由，本月已经跑过了，
#     再跑一遍会重复触发事件。**章节剧情结束后一律回 行动菜单。**
#   ↑ 为什么不是 chapter 主循环？chapter 会清空调用栈；虽然这里栈是空的，
#     但用 jump 语义更准确："我只是去演了一段，人还在主循环里。"
# ============================================================

# ---------- 伞下的选择：这一段定"路线" ----------
# ★ 路线用一个整数 flag 表示，而不是三个布尔：
#   路线 1=恋爱线 / 2=挚友线 / 3=沉默线
#   好处：结局判定只要 if 路线==1，不用写 `if 恋爱 && !挚友 && !沉默` 这种噩梦。
#
# ⚠⚠ 重要限制（HowToUse.md 里写错了，以这里为准）：
#   选项的 `flag:` 只吃**一个不含空格的 token**，因此只支持三种形式：
#       flag:名字      → 置 1
#       flag:名字+3    → 加 3
#       flag:名字-2    → 减 2
#   **不支持 `flag:路线 2` 这种赋任意值**（会被当成选项文字的一部分）。
#   要赋任意值，就像下面这样：选项只负责 `-> 标签`，把 `flag 路线 2`
#   写成目标 label 里的一条普通命令。多写一行，换来永远不会踩坑。
