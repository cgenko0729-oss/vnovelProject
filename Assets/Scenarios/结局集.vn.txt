# ============================================================
# 结局集.vn.txt —— 六种结局 + 结局判定表
# ------------------------------------------------------------
# 【进入方式】两条路：
#   ① 主循环 月初：if 剩余月数<=0 jump 结局集::结局判定   （时间耗尽）
#   ② 第3章 收尾：  jump 结局集::结局判定                  （告白演完）
#   两条路都用 jump 而不是 chapter —— 因为它们都是"流程的终点"，
#   不需要目标文件的头部初始化，也不需要清栈（此时栈本来就是空的）。
#
# 【本文件教学的机制】★ 判定表模式
#   结局判定 = 一串从"最难"排到"最容易"的 if，最后兜底一个默认结局。
#   **顺序就是优先级**，这是整个模式的全部秘密：
#     写在上面的条件更严格 → 先命中 → 玩家拿到"更高级"的结局。
#   兜底那一行不写 if，保证任何状态都有结局可演（永远不会跑飞）。
#
# 【为什么结局要单独一个文件】
#   ① 加结局不用碰任何其他文件
#   ② 所有"这一年到底算成功还是失败"的标准集中在一屏里，方便调平衡
#   ③ 第3章 只负责把结果存进 表白结果，不关心结局怎么演（关注点分离）
#
# 【判定用到的变量总览】（全都是普通 flag，存档自动带走）
#   表白结果  3=最好 2=成功 1=保留 0=没表白 -1=被拒
#   路线      1=恋爱 2=挚友 3=沉默
#   好感度_结衣 / 智力 / 魅力 / 压力 / 宅度
#   结衣知道我是宅 / 文化祭优胜 / 收到本命巧克力 / 看过她的软弱 / 有她的照片
# ============================================================


# ------------------------------------------------------------
# 结局判定表 —— 从最难到最容易，顺序即优先级
# ------------------------------------------------------------
label 结局判定
reset effects
letterbox on time:1.2
bgm stop fade:2

# ① 真结局：表白最高评价 + 她完全接受了"宅"的我 + 一年的伏笔全中
if 表白结果==3 && 结衣知道我是宅 && (收到本命巧克力 || 有她的照片) jump 结局_真
# ② 恋人结局：告白成功
if 表白结果>=2 jump 结局_恋人
# ③ 保留结局：她没给答案，但门没关上
if 表白结果==1 jump 结局_保留
# ④ 挚友结局：没告白（或被拒），但关系很深
if 好感度_结衣>=50 || 路线==2 || 看过她的软弱 jump 结局_挚友
# ⑤ 升学结局：感情线荒废，但把自己练成了另一个人
if 智力>=90 && 好感度_结衣<35 jump 结局_升学
# ⑥ 兜底：什么都没发生的一年（**这一行绝对不能加 if**）
jump 结局_平凡

星野结衣 微笑: 喂——加班到几点啊！我做好饭了哦！

星野结衣 微笑: 果然还是你最懂那种"世界上只有自己格格不入"的心情呢。
我: ……因为那本书的主角，就是当年的我啊。
