# ============================================================
# 好感事件_星野结衣.vn.txt —— 好感阈值触发的角色事件集
# ------------------------------------------------------------
# 【进入方式】主循环在两个地方 call 它：
#   月初一次、每次行动结束后一次 → call 好感事件_星野结衣::触发检查
#   用 call 而不是 jump：事件演完自动回到主循环，主循环完全不需要知道
#   里面发生了什么。**新增/删除角色事件时，主循环一行都不用改。**
#
# 【本文件教学的机制】★ 阶梯式触发器（进度指针模式）
#   一个整数 flag「结衣事件进度」当指针，事件按顺序解锁：
#       进度 0 + 好感>=25  → 事件1，演完把进度设为 1
#       进度 1 + 好感>=45  → 事件2，演完把进度设为 2
#       ...
#   为什么用"进度指针"而不是给每个事件一个 已看 flag：
#     ① 天然保证顺序（不会好感暴涨时一口气弹三段戏）
#     ② 一个变量就够，存档小、调试好看
#     ③ 想插入新事件只要改阈值，不用新开变量
#
# 【关键约束】
#   触发检查 的每一条出口都必须 return —— 包括"什么都没触发"那条。
#   漏掉 return 会带着调用栈跑到文件末尾 → 报错停机。
# ============================================================


# ------------------------------------------------------------
# 触发检查 —— 主循环唯一需要知道的入口
# ------------------------------------------------------------
label 触发检查
# 从高到低判断（先写高阈值），保证好感一次涨很多时也只触发"下一个"事件
if 结衣事件进度==3 && 好感度_结衣>=75 && 结衣知道我是宅 jump 事件4_同人展
if 结衣事件进度==2 && 好感度_结衣>=60 jump 事件3_深夜通话
if 结衣事件进度==1 && 好感度_结衣>=45 jump 事件2_便当
if 结衣事件进度==0 && 好感度_结衣>=25 jump 事件1_耳机
# ★ 没有任何事件触发 —— 这条 return 绝对不能少
return


# ------------------------------------------------------------
# 事件1（好感 25）：分一只耳机
# ------------------------------------------------------------
label 事件1_耳机
flag 结衣事件进度 1
call 公共片段::换场 背景:教室 心情:Neutral 转场:NoiseDissolve
show 星野结衣 at:center expr:默认 with:FadeSlideUp
bgm play 青春 fade:1.5

: 午休。她突然把一只耳机塞到我手里。
星野结衣 微笑: 喏，听听看！最近超上头的一首。
: （……耳机线只有那么长。也就是说，我们必须靠得很近。）

fx dof on
choice
* 老实凑过去听 -> 耳机_凑近
* 找借口拒绝 -> 耳机_拒绝

label 耳机_凑近
fx heartbeat on
: （她的头发擦到我的肩膀。洗发水的味道。）
: （歌是什么，我一个音都没听进去。）
星野结衣 微笑: 怎么样？好听吧！
我: ……嗯。非常，好听。
fx heartbeat off
call 公共片段::好感反馈 数值:10
jump 事件1收尾

label 耳机_拒绝
我: ……我、我耳朵不太舒服。
emote 星野结衣 Dejected
星野结衣 沮丧: 诶——真扫兴。
call 公共片段::好感反馈 数值:2
stat 压力 +10
jump 事件1收尾

label 事件1收尾
fx dof off
hide 星野结衣 with:fade
reset effects
# ★ 事件的出口也必须 return（回到 call 我的那一行的下一行）
return


# ------------------------------------------------------------
# 事件2（好感 45）：多做的那份便当
#   演示：event result 用在**剧情事件**里，不只是日程结算才能用
# ------------------------------------------------------------
label 事件2_便当
flag 结衣事件进度 2
call 公共片段::换场 背景:天台 心情:Sunset 转场:CircleWipe
fx godrays on
show 星野结衣 at:right expr:害羞 with:ShineReveal

星野结衣 害羞: 那个……我今天多做了一份便当。
星野结衣 害羞: 不、不是特地给你做的哦！只是分量算错了！
: （盒子里的玉子烧，形状歪得非常有诚意。）

# rand 分级：连"吃便当"这种小事都能做出随机性
flag 运气 rand:1-100
if 运气<=25 jump 便当_惨剧
if 运气<=80 jump 便当_普通

* 面不改色地全部吃完 -> 便当_硬吃
* 老实说"这个有点甜" -> 便当_老实

label 便当_硬吃
: （我一口气吃完了。眼泪是被甜出来的。）
我: ……很好吃。
星野结衣 惊讶: 骗人吧！我刚尝了一口差点吐出来！
星野结衣 害羞: ……但是，谢谢你。你这人真的，好奇怪。

sakura
: （……好吃。好吃到有点想哭。）
我: 你、你要不要考虑开店？
星野结衣 害羞: 笨蛋！哪有人这样夸的啦！
call 公共片段::好感反馈 数值:18

# ------------------------------------------------------------
# 事件3（好感 60）：凌晨两点的电话

# ------------------------------------------------------------

bgm play 忧郁 fade:2

: 凌晨两点。手机在桌上震动。是她。

我: 没有。怎么了。

星野结衣 沮丧: 没什么啦。就是……有点睡不着。
星野结衣 沮丧: 大家都觉得我很开朗对吧？每天都超有精神那种。
