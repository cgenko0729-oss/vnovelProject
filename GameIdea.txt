# 一、角色生命感

  这是我认为当前最值得投资的视觉方向。

  ## 1. 自动眨眼

  每个角色维护独立眨眼节奏：

  - 普通状态随机每 3～7 秒眨眼。
  - 紧张时眨眼更频繁。
  - 害羞时可以短暂闭眼。
  - 悲伤时缓慢闭眼、稍后睁开。
  - 惊讶动作触发时中断普通眨眼。

  最好使用独立眼睛层；如果素材没有分层，也可以准备普通、闭眼两张表情图快速交替。

  关键是加入少量随机性，不能所有角色同时眨眼。

  ## 2. 简易口型

  不必立即上 Live2D。可以先做三态口型：

  闭嘴 → 半张 → 张嘴

  驱动方式可以逐步升级：

  - 第一版：打字期间随机切换。
  - 第二版：语音播放期间根据音量振幅切换。
  - 第三版：读取语音时间轴中的口型标记。

  说完后必须回到闭嘴状态。旁白、内心独白和离场角色不驱动口型。

  ## 3. 视线与眼球移动

  角色的眼神可以根据当前说话者变化：

  - 看向另一位角色。
  - 看向玩家。
  - 害羞时向下或侧面移开。
  - 思考时看向上方。
  - 重要台词前先看向对方，再开口。

  即使眼睛只能移动 3～8 像素，也会明显增加生命感。

  ## 4. 微姿态变化

  在现有悬浮和呼吸上增加非常小的：

  - 头部倾斜。
  - 身体前倾或后退。
  - 肩部起伏。
  - 惊讶时缩一下。
  - 愤怒时身体向前压。
  - 犹豫时轻微左右摇摆。

  这些动作幅度要小，避免角色像果冻。

  ## 5. 角色距离层级

  允许角色存在不同舞台深度：

  - 前景角色：更大、更亮、更清晰。
  - 中景角色：正常。
  - 后景角色：略小、略暗、轻微模糊。

  配合角色走近、退后：

  moveDepth 黄黄 near 0.8
  moveDepth 黄黄 back 0.8

  这比单纯横向移动更有空间感。

  ## 6. 情绪表情层

  以后如果素材允许，可以把表情拆成独立图层：

  - 眼睛
  - 嘴
  - 眉毛
  - 腮红
  - 眼泪
  - 汗滴
  - 阴影

  这样“微笑＋闭眼＋腮红”可以组合，不需要为每种组合准备完整立绘。

  ## 7. 无立绘说话提示

  当角色在画面外说话时，可以：

  - 从对应屏幕边缘出现小头像。
  - 名牌旁增加方向箭头。
  - 声音做轻微左右声像偏移。
  - 背景边缘出现非常淡的角色主题色。

  # 二、台词与文字演出

  ## 1. 台词内嵌演出标记

  这是项目路线图里已经提到、现在很适合完成的功能。

  例如：

  黄黄: 我只是……{w:0.35}{shake:light}有一点害怕。
  黄黄: {speed:slow}如果明天以后……再也见不到你呢？

  建议支持：

  - {w:0.3}：句中停顿。
  - {speed:slow|normal|fast}：局部打字速度。
  - {shake:light}：文字或画面震动。
  - {se:心跳}：播放音效。
  - {emote:Shy}：说到这里才触发角色动作。
  - {color:#...}：关键词颜色。
  - {glow}：关键词短暂发光。
  - {size:120%}：强调文字。
  - {voice:v001}：本句语音。
  - {cam:预设名}：触发镜头预设。
  - {clear}：清空当前文本继续显示。

  这样演出不再只能发生在整句台词之前或之后。

  ## 2. 标点感知的打字节奏

  当前逐字速度如果完全相同，会比较机械。建议：

  - 逗号自动多停顿约 0.08～0.15 秒。
  - 句号、问号、感叹号停顿更长。
  - 省略号逐点出现并有特殊节奏。
  - 英文单词不要在单词中间产生奇怪停顿。
  - 玩家催促时立即完成，但不能重复播放嵌入音效。

  ## 3. 多种文本模式

  除了当前 ADV 对话框，还可以支持：

  - NVL 模式：大量旁白铺满屏幕。
  - 手机聊天模式：气泡消息逐条出现。
  - 内心独白模式：无名牌、斜体或特殊边框。
  - 信件/日记模式：纸张全屏 UI。
  - 系统提示模式：居中小字、无对话框。
  - 章节标题模式：大标题＋副标题＋日期地点。
  - 电影字幕模式：隐藏对话框，只显示底部字幕。

  ## 4. 文字高级排版

  可以加入：

  - 注音/Ruby Text。
  - 人名、地名、世界观术语自动高亮。
  - 不同角色使用轻微不同的文字音效。
  - 特殊角色使用独特名牌形状。
  - 恐惧台词的文字轻微抖动。
  - 虚弱台词逐渐降低透明度。
  - 电话中的声音使用不同字体颜色或滤镜。

  ## 5. 回想界面升级

  现有 Backlog 可以进一步支持：

  - 点击某句重播语音。
  - 显示说话者头像。
  - 显示该句发生时的背景缩略图。
  - 搜索角色或关键词。
  - 标记重要句子。
  - 从某句创建书签。
  - 在允许的范围内回滚到该句。

  # 三、音频系统深化

  你已有 BGM 交叉淡化、SE、Voice 和音量控制，下一步是“声音导演”。

  ## 1. AudioMixer 分组

  建议分为：

  Master
  ├─ BGM
  ├─ Voice
  ├─ SE
  ├─ Ambience
  └─ UI

  这样玩家可以分别控制环境声和 UI 声，而不是所有音效共用一个滑条。

  ## 2. 语音驱动 Auto

  Auto 等待时间不应只按字数计算：

  - 有语音：等待语音播放结束＋0.3～0.8 秒。
  - 无语音：继续使用字数计算。
  - 玩家点击时可选择“立即完成文字”或“同时停止语音”。
  - 设置中提供“点击下一句时停止上一句语音”开关。

  ## 3. BGM 自动 Ducking

  角色语音开始时：

  - BGM 自动降低约 2～5 dB。
  - 语音结束后缓慢恢复。
  - 爆炸、雷声等重要 SE 也可以短暂压低 BGM。

  这个改动不显眼，但会让声音清晰专业很多。

  ## 4. 环境声分层

  不要只播放一个“雨声”循环。可以拆成：

  - 持续雨声底层。
  - 随机屋檐滴水。
  - 远处雷声。
  - 偶尔车辆经过。
  - 室内窗户的低频震动。

  夜晚场景则可以有虫鸣、风声、远处列车。随机的一次性声音会让环境不再像重复音频循
  环。

  ## 5. 场景声学预设

  不同地点使用不同混响和滤波：

  - 室内：轻微短混响。
  - 大礼堂：长混响。
  - 电话：削低频和高频。
  - 梦境：延迟、混响、低通。
  - 回忆：高频衰减＋轻微噪声。
  - 水下：强低通＋气泡声。

  ## 6. 音乐节拍联动

  重要镜头尽量对齐：

  - 音乐小节开始。
  - 鼓点。
  - 副歌进入。
  - 音乐停顿。

  可以给 BGM 配少量时间标记，在特定时间触发白闪、切镜、标题或角色登场。

  # 四、镜头与导演系统继续提升

  ## 1. VNDirector 复合演出预设

  你现在有很多底层能力，应该把它们组合成语义化导演命令：

  director confession intensity:0.8
  director suspicion target:黄黄
  director memory enter
  director thunder
  director emotionalBreakdown target:黄黄

  例如“告白”可以组合：

  - BGM 压低。
  - 心跳开始。
  - 黄昏色调加强。
  - 缓慢推向角色。
  - 角色害羞动作。
  - 关键词出现时樱花爆发。
  - 结束后平滑恢复。

  重要的是每个预设接受强度和时长参数，而不是写死。

  ## 2. 演出强度总线

  建立全局强度概念：

  calm / normal / tense / climax

  它可以同时控制：

  - 镜头速度。
  - Bloom 强度。
  - 角色呼吸幅度。
  - 粒子密度。
  - 边缘泛光。
  - BGM 压低程度。
  - 震动幅度。

  这样同一个 director confession 可以在安静告白和激烈告白中复用。

  ## 3. 镜头语法一致性

  建议为作品制定固定语言：

  - 缓推：亲密、压力、认真。
  - 急推：震惊或发现。
  - 硬切：冲突、打断、现实冲击。
  - 交叉叠化：回忆、时间流逝、温柔视线转移。
  - 缓慢横移：观察、犹豫、空间关系。
  - Dolly Zoom：只用于极少数认知崩塌场面。
  - 眨眼转场：角色主观意识变化。
  - 白闪：物理冲击、强烈回忆或极端情绪。

  一致使用以后，玩家会在潜意识里读懂镜头。

  ## 4. 镜头连续性检查

  编辑器可以提示：

  - 连续两个镜头缩放差异太大。
  - 新镜头可能露边。
  - 角色从画面左侧无理由跳到右侧。
  - 过短时间内连续多个急推。
  - 对话进行中截图叠化可能冻结文字。
  - 镜头结束没有复位或保存状态。

  ## 5. 镜头安全区预览

  镜头编辑器增加：

  - 16:9、16:10、21:9、安全区域切换。

  # 五、背景与场景表现

  ## 1. 多层背景导入工作流

  把一张背景拆成：

  天空
  远景
  建筑
  中景
  前景树叶/窗框
  局部光效

  不同层使用不同视差速度。前景遮挡物尤其重要，它会让角色像站在环境里，而不是贴在背
  景上。

  ## 2. 动态场景小物件

  高性价比动态元素：

  - 窗帘轻摆。
  - 树叶晃动。
  - 电风扇转动。
  - 灯光微闪。
  - 水面反光。
  - 蒸汽上升。
  - 电视画面变化。
  - 雨滴沿玻璃滑落。
  - 远处车辆灯光经过。
  - 教室时钟秒针移动。

  每个场景只需一两个，就能明显增加活力。

  ## 3. 时间变化

  同一背景支持：

  - 清晨。
  - 正午。
  - 黄昏。
  - 夜晚。
  - 室内开灯。
  - 暴雨前。
  - 雨后湿润。

  可以通过色调、天空叠层、窗户发光、阴影方向和环境声组合完成，不一定要准备完整新背
  景。

  ## 4. 天气与环境互动

  当前天气主要是粒子层，可以继续加入：

  - 雨落到地面产生涟漪。
  - 角色站在雨中时立绘出现轻微湿润高光。
  - 雷电瞬间照亮背景和角色边缘。
  - 雪在角色头发/肩膀上少量积累。
  - 风吹动前景叶子和角色头发装饰。
  - 萤火虫靠近角色时照亮局部轮廓。

  ## 5. 自动角色光照匹配

  现在有色调匹配，可以继续加入：

  - 根据背景平均亮度调整角色亮度。
  - 黄昏时增加暖色边缘光。
  - 夜景时降低角色正面亮度。
  - 窗边场景根据窗户方向产生单侧光。
  - 雷电时角色产生短促冷色轮廓光。

  # 六、视觉小说核心玩法与收集

  ## 1. Rollback

  允许玩家回到前几句并恢复：

  - 角色表情。
  - 背景。
  - 音乐。
  - 镜头。
  - Flags。
  - 选择前状态。

  这比只看 Backlog 更符合视觉小说习惯，但实现难度明显高于普通回想，需要状态历史或

  ## 2. 只跳过已读内容

  Skip 建议分为：

  - 只跳过已读。
  - 跳过全部。

  ## 3. 路线流程图

  提供可选的路线图：

  - 已到达节点亮起。
  - 未探索路线隐藏或显示问号。

  建议包括：

  - CG Gallery。
  - Ending Gallery。
  - BGM Music Room。
  - Scene Replay。
  - Voice Collection。
  - 角色资料。
  - 世界观词典。

  场景回放可以直接利用现有脚本标签和镜头预设。

  ## 5. 好感度与变量表现

  不一定直接显示数字，可以使用：

  - 角色资料页的花瓣数量。
  - 名牌边框逐渐变化。
  - 标题画面角色距离变化。
  - 存档缩略图上的关系图标。
  - 路线图中的颜色变化。

  这样比直接显示“好感度 +2”更有沉浸感。

  ## 6. 选择演出深化

  不同选择类型可以有不同表现：

  - 普通选择：稳定按钮。
  - 紧急选择：倒计时，但必须允许关闭时间限制。
  - 道德抉择：背景去饱和、声音低通。
  - 隐藏选择：满足条件后额外选项缓慢显现。

  # 八、可访问性与舒适性

  这是正式作品很容易忽略、但非常值得早做的部分。

  建议设置：

  - 字体大小、行距、字距。
  - 对话框透明度。
  - 高对比度文字背景。
  - 可选更易读的无衬线字体。
  - 禁止屏幕震动。
  - 降低镜头移动。
  - 禁止爆闪。
  - 降低 Bloom。
  - 降低粒子密度。
  - 禁止鼠标视差。
  - 自动播放无需长按。
  - 选择不设置时间限制。
  - 音效字幕，例如“[远处雷声]”“[门被推开]”。
  - 重要信息不能只依靠颜色表达。

  微软当前的游戏可访问性指南将文字、对比度、字幕、输入、UI
  导航、动态效果和光敏风险分别列为核心项目，非常适合用作检查表：Xbox Accessibility
  Guidelines (https://learn.microsoft.com/en-us/xbox/accessibility/guidelines)。

  尤其你的项目有 Bloom、白闪、震动、粒子和大量镜头运动，因此“减少动态”和“减少闪
  光”应该成为正式设置，而不是删除这些漂亮演出。

  # 十一、性能和项目架构

  ## 1. 控制透明过绘

  项目大量使用：

  - 全屏透明 UI。
  - Bloom。
  - 多层粒子。
  - 模糊。
  - 多张全屏截图叠化。

  这些都是 GPU 过绘热点。建议提供性能等级：

  - 高：全部效果。
  - 中：降低粒子、模糊采样和 Bloom。
  - 低：关闭昂贵的热浪、模糊、云影叠层。

  ## 2. 预加载下一段资源

  在玩家阅读当前台词时预加载：

  - 下一张背景。
  - 即将出现的角色表情。
  - 下一段语音。
  - 即将播放的 BGM。
  - 镜头叠化所需纹理。

  这样可以避免换背景或播放语音时卡顿。

  ## 3. Addressables 的使用时机

  如果最终只有少量章节，不必急着迁移。但当背景、CG、全语音和多语言音频大量增长后，
  Addressables
  才值得加入，用于章节资源分组、异步加载和远程内容更新。官方文档确认远程目录可以在
  不重新发布完整应用的情况下更新远程资源：Unity Addressables 远程内容说明
  (https://docs.unity3d.com/kr/Packages/com.unity.addressables%401.21/manual/remote-content-intro.html)。

  - 当章关键场景短暂叠化。
  - 重要台词以声音回响。

  - 最后显示章节标题或日期。

  - 坏结局后标题音乐少一个乐器声部。
